# Lab2：DNS 初体验，域名是怎么变成 IP 的？

## 实验背景

当我们在浏览器中输入 `www.yxnu.edu.cn` 这样的域名时，计算机并不能直接依靠这个名字找到目标主机，它首先需要通过 **DNS（Domain Name System，域名系统）** 把域名解析成 IP 地址，然后才能继续访问目标服务器。

一次典型的 DNS 解析过程可以简单理解为：

```text
应用程序 ──── 发起域名解析请求 ────▶ DNS 服务器
应用程序 ◀──── 返回域名对应 IP ──── DNS 服务器
```

本实验中，我们使用 C 语言调用系统提供的域名解析接口，查询某个域名对应的 IP 地址；同时使用 Wireshark 抓取 DNS 请求包和响应包，观察域名解析在网络中的实际表现。

> **说明**：本次实验为了降低难度，使用 `gethostbyname()` 完成解析。  
> 这个函数是较老的接口，不支持 IPv6，线程安全性也较差，在实际工程中通常更推荐使用 `getaddrinfo()`。  
> 但在入门阶段，`gethostbyname()` 更容易帮助大家理解“域名解析”这件事是如何发生的。

## 参考代码

将下列代码保存为 `dns_lookup.c`，然后编译运行。

```c
#include <stdio.h>
#include <netdb.h>
#include <arpa/inet.h>

int main()
{
    struct hostent *host = gethostbyname("www.yxnu.edu.cn");
    if (!host)
    {
        herror("查询失败");
        return 1;
    }
    printf("IP: %s\n", inet_ntoa(*(struct in_addr *)host->h_addr));
    return 0;
}

```

编译与运行示例：

```bash
gcc dns_lookup.c -o dns_lookup
./dns_lookup 
```

## 思考题

1. DNS 的作用是什么？为什么访问网站时通常要先进行 DNS 解析？

   > 答：NS（域名系统）是互联网的核心基础设施，核心作用是将人类易读的域名（如www.baidu.com）转换为机器可识别的 IP 地址，同时也可实现反向解析（IP 转域名）、记录别名、邮件服务器定位等功能。
   网络设备间的通信依赖 IP 地址标识目标节点，而普通用户无法记忆复杂的 IP 地址。通过 DNS 解析，可将易记的域名映射为实际 IP，让浏览器 / 应用层程序能与目标服务器建立网络连接，是访问网站的前置必要步骤。

2. 你抓到的 DNS 请求包和响应包中的 **Transaction ID** 是否一致？这个字段有什么作用？

   > 答：本次实验中，DNS 请求包（0x4c40）和响应包的Transaction ID 完全一致。
字段作用：该字段是 16 位的唯一标识，用于匹配 DNS 请求与对应的响应。网络中可能存在多个 DNS 请求，接收端通过该 ID 可快速将响应归属于对应的请求，避免报文混淆，同时也能检测请求是否超时、响应是否异常。

3. 这次实验中查询类型通常是 `A` 记录。`A` 记录表示什么含义？

   > 答：A记录（Address Record，类型值为 1）是 DNS 中最基础的资源记录类型，用于将主机名（域名）映射为 IPv4 地址。

4. DNS 默认通常使用哪个传输层协议和端口？在什么情况下可能会改用 TCP？

   > 答：DNS 默认使用UDP 协议，端口为53
   响应报文大小超过 UDP 的最大限制
进行区域传输
查询请求为DNSSEC相关的安全记录


5. 为什么说 `gethostbyname()` 不是现代工程中推荐继续使用的接口？请至少写出两点原因。

   > 答：仅支持 IPv4：该接口只能处理 IPv4 地址，无法满足现代 IPv6 网络的需求，存在明显的协议兼容性缺陷。
线程不安全、不可重入：接口使用全局静态缓冲区存储解析结果，多线程环境下并发调用会导致数据竞争、结果覆盖，线程安全问题严重。
阻塞式调用：属于同步阻塞接口，解析过程中会阻塞当前进程 / 线程，无法实现异步解析，在高并发、高性能网络应用中效率极低。
废弃与标准淘汰：已被 POSIX 标准标记为废弃，现代推荐使用

6. 如果一个域名返回了多个 IP 地址，这通常说明了什么？

   > 答：实现负载均衡：服务器将多个 IP 对应不同的后端物理服务器，通过 DNS 轮询（Round Robin）等策略，将用户请求分发到不同服务器，分摊访问压力，提升系统吞吐量。
服务高可用：多个 IP 对应不同的服务器节点，若某一节点故障，DNS 可将请求指向其他正常节点，避免服务中断，保障业务连续性。
CDN 加速需求：如百度等网站通过 CDN 部署，一个域名会对应多个地域 / 节点的 IP，实现就近访问，提升访问速度。
